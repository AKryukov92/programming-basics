Задача

Переопределите стандартный метод GetHashCode для расчета хеша экземпляра класса Point

  1. В классе Point реализуйте публичный метод GetHashCode с аннотацией @Override.
  2. Метод GetHashCode не принимает аргументов и возвращает целое число.
  3. В теле метода верните значение, вычисляемое по формуле x+31y.

Проверьте, что хеши одинаковых экземпляров одинаковы

  1. Добавьте в класс TestSuite юнит-тест SamePointsWithSameHashcode. В нем создайте 3 экземпляра класса Point с координатами (0;0).
  2. Инициализируйте поле color двух из них значением "green", а третьего - значением "red".
  3. Проверьте, что у всех этих точек вызов метода GetHashCode возвращает одинаковый результат.
  4. Проверьте, что у всех комбинаций из двух точек вызов метода Equals возвращает истину.
  5. После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.

Проверьте, что дубликат точки не добавляется в set

  1. Добавьте в класс TestSuite юнит-тест HashSetContainsOnlyOneOfSamePoints.
  2. Создайте коллекцию pts типа HashSet для экземпляров класса Point.
  3. Добавьте в pts с помощью метода add 2 экземпляра класса Point с координатами (0;0) и цветом "red".
  4. Добавьте в pts с помощью метода Add экземпляр класса Point с координатами (0;0) и цветом "green".
  5. Проверьте, что свойство Count коллекции pts возвращает 1.
  6. Проверьте, что коллекция pts содержит точку с координатами (0;0) и цветом "red". Для этого вызовите метод contains коллекции pts и передайте ему экземпляр точки в качестве аргумента. Результат вызова метода передайте в качестве аргумента методу IsTrue класса Assert.
  7. После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.
  8. Посмотрите с помощью отладчика, что у единственного элемента множества pts в поле color содержится значение "red".

Проверьте, что хеши различных экземпляров, как правило, различны

  1. Добавьте в класс TestSuite юнит-тест DifferentPointsWithDifferentHashcode. В нем создайте несколько экземпляров класса Point с координатами (0;0), (0;5), (5;0), (5;5) не указывая цвет.
  2. Проверьте, что вызовы метода GetHashCode у этих экземпляров возвращают различные значения.
  3. Проверьте, что у всех комбинаций из двух точек вызов метода Equals возвращает ложь.
  4. После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.

Проверьте, что хеши различных экземпляров могут быть одинаковы

  1. Добавьте в класс TestSuite юнит-тест DifferentPointsWithSameHashcode. В нем создайте два экземпляра класса Point с координатами (0;1) и (31;0).
  2. Проверьте, что у всех этих точек вызов метода GetHashCode возвращает одинаковый результат.
  3. Проверьте, что вызов метода Equals возвращает ложь.
  4. После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.

Проверьте, что различные экземпляры с различными хешами успешно добавляются в set

  1. Добавьте в класс TestSuite юнит-тест HashSetContainsAllDifferentPointsWithDifferentHashcode. В нем создайте несколько экземпляров класса Point с координатами (0;0), (0;5), (5;0), (5;5) не указывая цвет.
  2. Создайте коллекцию pts типа HashSet для экземпляров класса Point.
  3. Добавьте экземпляры класса Point в pts с помощью метода add.
  4. Проверьте, что свойство Count коллекции pts возвращает 4.
  5. Проверьте, что коллекция pts содержит точки с координатами (0;0), (0;5), (5;0), (5;5).
  6. После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.

Проверьте, что различные экземпляры с одинаковыми хешами успешно добавляются в set

  1. Добавьте в класс TestSuite юнит-тест hashSetContainsAllDifferentPointsWithSameHashcode. В нем создайте два экземпляра класса Point с координатами (0;1) и (31;0).
  2. Создайте коллекцию pts типа HashSet для экземпляров класса Point.
  3. Добавьте экземпляры класса Point в pts с помощью метода add.
  4. Проверьте, что свойство Count коллекции pts возвращает 2.
  5. Проверьте, что коллекция pts содержит точки с координатами (0;1), (31;0).
  6. После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.
