Задача

  1. В файле для исходного кода "lab13.js" реализуйте функцию task3943 для решения задачи ЛР2#3943 (открыть в новой вкладке).
  2. В теле функции task3943 нужно предусмотреть обработку некорректных исходных данных.
  3. Если в качестве аргументов были переданы некорректные значения, нужно создать исключение и "выбросить" его с помощью ключевого слова throw. При создании исключения, в скобках нужно указать текст ошибки.
  4. Если все аргументы корректны, нужно вычислить результат и вернуть его.
  5. В файле для юнит-тестов "lab13spec.js" проверьте с помощью юнит-теста результат работы функции task3943 в случаях с корректными данными.
  6. Проверьте с помощью вспомогательного файла Runner3943.html результат работы функции task3943 в случаях с корректными данными.
-10,000000 -435,706230

Проверьте юнит-тестами, что функция выбрасывает исключение при некорректных данных

  1. Чтобы убедиться, что функция выбрасывает исключение, нужно воспользоваться функцией expect в сочетании с функцией toThrow.
  2. В теле юнит-теста создайте экземпляр ошибки, который будет "эталоном" для сравнения.
  3. Сначала нужно вызвать функцию expect и передать ей в качестве аргумента анонимную функцию, в теле которой нужно вызвать функцию task3943 с некорректным значением аргумента.
  4. В конце той же строки нужно поставить точку и вызвать функцию toThrow. Единственный ее аргумент - эталон ошибки.
  5. Всего таких конструкций должно получиться 2. По одной для каждого некорректного значения.
  6. При открытии файла SpecRunner.html, в нем должна быть зеленая строка "в задаче 3943 должен получиться корректный результат".

Добавьте во вспомогательный файл Runner3943.html обработку исключений

  1. Вызовите функцию task3943 с аргументами из каждого теста, указанного в задаче.
  2. В случае успешного выполнения функции task3943, выведите в консоль результат с точностью до 4 знаков после запятой.
  3. В случае возникновения исключения, нужно его перехватить и вывести в консоль сообщение об ошибке.
  4. При открытии файла Runner3943.html, в консоли должен появиться текст:
-10,000000 -435,706230 Значение y должно быть неотрицательным Подкоренное выражение должно быть неотрицательным
