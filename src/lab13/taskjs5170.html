Задача

  • В файле для исходного кода "lab13.js" реализуйте функцию task5170 для решения задачи ЛР6#5170 (открыть в новой вкладке).
  • В теле функции task5170 нужно предусмотреть обработку некорректных исходных данных.
  • Если в качестве аргументов были переданы некорректные значения, нужно создать исключение и "выбросить" его с помощью ключевого слова throw. При создании исключения, в скобках нужно указать текст ошибки.
  • Если все аргументы корректны, нужно вычислить результат и вернуть его.
  • В файле для юнит-тестов "lab13spec.js" проверьте с помощью юнит-теста результат работы функции task5170 в случаях с корректными данными.
  • Проверьте с помощью вспомогательного файла Runner5170.html результат работы функции task5170 в случаях с корректными данными.
  • Если вызов функции task5170 вернул true, выведите в консоль фразу "число простое".
  • Если вызов вернул false, то выведите "число не простое".

Проверьте юнит-тестами, что функция выбрасывает исключение при некорректных данных

  • Чтобы убедиться, что функция выбрасывает исключение, нужно воспользоваться функцией expect в сочетании с функцией toThrow.
  • Должна получиться всего 1 проверка возникновения исключения.
  • При открытии файла SpecRunner.html, в нем должна быть зеленая строка "в задаче 5170 должен получиться корректный результат".
