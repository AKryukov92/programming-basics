Задача

  • В классе Lab06 реализуйте публичный статический метод task5170. Он принимает 1 целое число, возвращает логическое значение и может выбросить исключение java.lang.IllegalArgumentException.
  • В теле метода нужно вернуть результат решения задачи ЛР6#5170 (открыть в новой вкладке) используя значение аргумента в качестве исходных данных. В случае ошибки, нужно выбросить исключение IllegalArgumentException и указать при этом текст ошибки из задачи.
  • В классе Lab06Suite создайте публичный нестатический метод task5170test. Он помечен аннотацией @Test, ничего не возвращает и не принимает аргументы.
  • В теле метода task5170test нужно 4 раз вызвать метод task5170 класса Lab06. При каждом вызове в качестве аргументов нужно передавать значения из разных тестов, которые приводят к вычислению результата.
  • Результат каждого вызова нужно сравнить с ожидаемым результатом с помощью статических методов assertTrue или assertFalse класса Assert. Единственный аргумент этих методов - результат вызова метода task5170.
  • После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.
  • Создайте публичный нестатический метод, который ничего не возвращает, не принимает аргументов и называется task5170assertion. Пометьте его аннотацией @Test с элементом expected равным IllegalArgumentException.class.
  • В теле метода task5170assertion нужно вызвать метод task5170 класса Lab06 и указать значения, которые приводят к выбрасыванию исключения.
  • После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.
  • В папке с кодом создайте класс Runner5170. В этом классе создайте метод main.
  • В методе main класса Runner5170 создайте массив inY. Заполните его значениями из тестов к задаче.
  • Вызовите метод task5170 класса Lab06 столько раз, сколько тестов в задаче. В каждом вызове передайте методу в качестве аргумента элемент массива inY. Поймайте потенциальное исключение IllegalArgumentException.
  • Если вызов метода task5170 вернул true, выведите в консоль фразу "число простое". Если вызов вернул false, то выведите "число не простое". В случае поимки исключения, выведите в консоль сообщение об ошибке.
  • В результате запуска метода main класса Runner5170, в консоли должен появиться текст.
