У вас должен получиться код, похожий на фрагмент ниже.

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Example { class Program { static void Main(string[] args) { String data; data = Console.ReadLine(); double x; x = double.Parse(data); double result; Console.Write("Результат: {0:F4}", result); } } }

Следующим шагом нужно оформить вычисление результата. Команда выглядит вот так:

result = x + 7;

Выражение "result = ..." как и раньше это "присвоить переменной result значение ...". А значением тут является результат вычисления арифметического выражения "К значению переменной x прибавь число 7".

Предположите что получится, если вычисление оформить так:

data = x + 7;

Или так:

result = data + 7;

Попробуйте написать такой код и запустить. Подтвердилось ли ваше предположение?

Шаг 2/2

К шагу 1 К вступлению

Вернитесь к первому предложенному варианту решения.

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Example { class Program { static void Main(string[] args) { String data; data = Console.ReadLine(); double x; x = double.Parse(data); double result; result = x + 7; Console.Write("Результат: {0:F4}", result); } } }

Отдельные части арифметического выражения названы терминами: операторы и операнды. Оператор это действие, а операнд - то над чем оно выполняется. Если отметить их цветом в решаемой задаче, то получится так:

result = x + 7; Строго говоря, "result" это тоже операнд, а "=" это тоже оператор.

Операторы бывают:
*умножение /деление +сложение -вычитание %остаток от деления
Умножение и деление вычисляются раньше сложения и вычитания. Одинаковые операторы вычисляются слева направо.

Операнды намного более разнообразны. Ими могут быть:

  • Литералы. Т.е. числа, написанные в коде.
  • Переменные.
  • Выражения в круглых скобках.
  • Вызовы методов. Такие операторы будут рассмотрены позже.

Выражения в круглых скобках нужны для указания порядка вычисления. Сравните:

//Вариант 1 result = (1 + 2) * 3; //Вариант 2 result = 1 + 2 * 3;

В первом случае результат будет 9, а во втором 7.

Как правило, операнды должны быть совместимого типа. Например "+" можно использовать для работы со строками, а "-", "*" и "/" нельзя. Строки и числа можно склеить в одну строку оператором "+", но оператор "*" в том же выражении приведет к синтаксической ошибке.

Как вы думаете, есть ли подобные ограничения при работе с целыми и дробными (действительными) числами?

Вступление

В предыдущем разборе теории (открыть в новой вкладке) был рассмотрен ввод чисел с клавиатуры.

Сейчас будет разобран пример оформления вычислений в программе.

Прежде чем отдавать компьютеру команду "посчитай результат", нужно понять назначение каждой переменной.

Какие переменные были объявлены в фрагменте программы ниже?

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Example { class Program { static void Main(string[] args) { String data; data = Console.ReadLine(); double x; x = double.Parse(data); } } }

Их всего две:

  • data - нужна чтобы получить значение от пользователя в текстовом виде.
  • x - нужна для хранения значения от пользователя в виде числа (действительного)

Для хранения результата будет полезно создать еще одну переменную. В абстрактной задаче ее можно назвать просто "result". Если же в задаче, к примеру, дата плотность тела и его объем, а нужно найти массу, то переменным нужно дать более осмысленные названия: density, volume, mass.

Добавьте команду переменную для результата и команду ее вывода в консоль с нужной точностью.
