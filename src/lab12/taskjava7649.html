Строки являются неизменными, поэтому частая их модификация приводит к созданию новых объектов, что в свою очередь расходует драгоценную память. Для решения этой проблемы были созданы классы java.lang.StringBuilder и java.lang.StringBuffer, которые позволяют более эффективно работать над модификацией строки. Классы является mutable, то есть изменяемым — используйте его, если Вы хотите изменять содержимое строки. StringBuilder был введен в Java 5 и имеет полностью идентичный API с StringBuffer. Единственное отличие — StringBuilder не синхронизирован. Это означает, что его использование в многопоточных средах есть нежелательным. Следовательно, если вы работаете с многопоточностью, Вам идеально подходит StringBuffer, иначе используйте StringBuilder, который работает намного быстрее в большинстве реализаций. Больше информации: по русски (в новой вкладке), по английски (в новой вкладке)
