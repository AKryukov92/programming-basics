Задача

Переопределите стандартный метод Equals для проверки эквивалентности двух экземпляров класса Point.

  1. В классе Point создайте публичный метод Equals и добавьте перед типом возвращаемого значения ключевое слово override.
  2. Он должен возвращать логическое значение и принимать в качестве аргумента экземпляр типа Object.
  3. В теле метода Equals верните значение false.

Проверьте, что экземпляр точки эквивалентен сам себе

  1. Добавьте юнит тест PointEqualsItself. В нем создайте экземпляр класса Point.
  2. Вызовите у экземпляра класса Point метод Equals и передайте в качестве аргумента этот же экземпляр.
  3. Проверьте результат вызова Equals с помощью метода IsTrue класса Assert.
  4. При запуске тестов, юнит-тест PointEqualsItself должен быть отмечен красным знаком.
  5. Укажите, что имя аргумента типа Object в методе Equals будет "other".
  6. Добавьте в тело метода Equals выражение:
    if (this == other) return true;
  7. Теперь PointEqualsItself при запуске должен быть отмечен зеленым знаком.

Проверьте, что эквивалентность точек зависит от значения их полей

  1. Добавьте юнит-тест PointInstanceComparison. В нем объявите переменные типа Point: "a1", "a2", "b1", "b2", "b3".
  2. Инициализируйте эти переменные экземплярами класса Point. Инициализируйте их поля значениями по таблице.
    Имя переменнойxy
    a1 10 15
    a2 10 15
    b1 0 0
    b2 10 3
    b3 1 15
  3. Проверьте с помощью метода IsTrue, что все комбинации вызова метода Equals для сравнения "a1" и "a2" возвращают истину. Всего 2 сравнения.
  4. Проверьте с помощью метода IsFalse, что все комбинации вызова метода Equals для сравнения каждой из точек "b" и каждой из точек "a" возвращают ложь. Всего 16 сравнений.
  5. При запуске тестов, юнит-тест PointInstanceComparison должен быть отмечен красным знаком.
  6. Добавьте в тело метода Equals выражение:
    Point point = (Point) other; if (x == point.x && y == point.y) return true;
  7. Теперь PointInstanceComparison при запуске должен быть отмечен зеленым знаком.

Проверьте, что сравнение со значением null возвращает false и не выбрасывает исключение NullPointerException

  1. Добавьте юнит-тест PointNotEqualNull.
  2. В нем объявите переменную типа Point с именем p. Инициализируйте ее экземпляром класса Point.
  3. Вызовите метод Equals и передайте ему в качестве аргумента значение null.
  4. Проверьте результат вызова Equals с помощью метода IsFalse.
  5. При запуске тестов, юнит-тест PointNotEqualNull должен быть отмечен красным знаком.
  6. Добавьте в тело метода Equals выражение:
    7. if (other == null) return false;
  7. Все тесты в классе PointSuite должны быть отмечены зеленым знаком.

Проверьте, что сравнение с экземпляром другого класса возвращает false и не выбрасывает никаких исключений

  1. Добавьте юнит-тест PointNotEqualOtherClass. В нем создайте экземпляр класса Point.
  2. Вызовите метод Equals и передайте ему в качестве аргументов строку "not a point".
  3. Проверьте результат вызова с помощью метода IsFalse
  4. При запуске тестов, юнит-тест PointNotEqualOtherClass должен быть отмечен красным знаком.
  5. Добавьте в тело метода Equals выражение:
    6. if (GetType() != other.GetType()) return false;
  6. Тест PointNotEqualOtherClass может быть провален из за возникшего исключения System.InvalidCastException. В этом случае нужно изменить порядок действий.
  7. Все тесты в классе PointSuite должны быть отмечены зеленым знаком.
