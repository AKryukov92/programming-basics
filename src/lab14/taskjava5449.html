Задача

Переопределите стандартный метод equals для проверки эквивалентности двух экземпляров класса Point.

  1. Убедитесь, что у класса Point объявлены поля для целочисленных значений "x" и "y".
  2. В классе Point создайте публичный метод equals и пометьте его аннотацией @Override.
  3. Он должен возвращать логическое значение и принимать в качестве аргумента экземпляр типа Object.
  4. В теле метода Equals верните значение false.

Проверьте, что экземпляр точки эквивалентен сам себе

  1. Добавьте юнит тест pointEqualsItself. В нем создайте экземпляр класса Point.
  2. Вызовите у экземпляра класса Point метод equals и передайте в качестве аргумента этот же экземпляр.
  3. Проверьте результат вызова equals с помощью метода assertTrue класса Assert.
  4. При запуске тестов, юнит-тест pointEqualsItself должен быть отмечен красным знаком.
  5. Укажите, что имя аргумента типа Object в методе Equals будет "other".
  6. Добавьте в тело метода equals выражение:
    if (this == other) return true;
  7. Теперь pointEqualsItself при запуске должен быть отмечен зеленым знаком.

Проверьте, что эквивалентность точек зависит от значения их полей

  1. Добавьте юнит-тест pointInstanceComparison. В нем создайте три экземпляра класса Point: "a1", "a2", "b1", "b2", "b3".
  2. Инициализируйте поля x и y точек "a1" и "a2" значениями 10 и 15.
  3. Инициализируйте поля x и y точки "b1" значениями 10 и 3, точки "b2" значениями 1 и 15, точки "b3" значениями 0 и 0.
  4. Проверьте с помощью метода assertTrue, что все комбинации вызова метода equals для сравнения "a1" и "a2" возвращают истину. Всего 2 сравнения.
  5. Проверьте с помощью метода assertFalse, что все комбинации вызова метода equals для сравнения каждой из точек "b" и каждой из точек "a" возвращают ложь. Всего 12 сравнений.
  6. При запуске тестов, юнит-тест pointInstanceComparison должен быть отмечен красным знаком.
  7. Добавьте в тело метода equals выражение:
    Point point = (Point) other; if (x == point.x && y == point.y) return true;
  8. Теперь pointEqualsItself при запуске должен быть отмечен зеленым знаком.

Проверьте, что сравнение со значением null возвращает false и не выбрасывает исключение NullPointerException

  1. Добавьте юнит-тест pointNotEqualNull. В нем создайте экземпляр класса Point.
  2. Проверьте результат вызова equals с помощью метода assertFalse.
  3. При запуске тестов, юнит-тест pointNotEqualNull должен быть отмечен красным знаком.
  4. Добавьте в тело метода equals выражение:
    5. if (other == null) return false;
  5. Все тесты в классе PointSuite должны быть отмечены зеленым знаком.

Проверьте, что сравнение с экземпляром другого класса возвращает false и не выбрасывает никаких исключений

  1. Добавьте юнит-тест pointNotEqualOtherClass. В нем создайте экземпляр класса Point.
  2. Вызовите метод equals и передайте ему в качестве аргументов строку "not a point".
  3. Проверьте результат вызова с помощью метода assertFalse
  4. При запуске тестов, юнит-тест pointNotEqualOtherClass должен быть отмечен красным знаком.
  5. Добавьте в тело метода equals выражение:
    6. if (getClass() != other.getClass()) return false;
  6. Все тесты в классе PointSuite должны быть отмечены зеленым знаком.
