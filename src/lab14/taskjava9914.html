Задача

Реализуйте НЕстатический метод для решения задачи ЛР5#6589 (открыть в новой вкладке)

  • Убедитесь, что у класса Range объявлены поля для целочисленных значений "from" и "to".
  • В классе Point реализуйте публичный нестатический метод intersects(без ключевого слова static перед типом возвращаемого значения).
  • Метод intersects принимает в качестве аргумента 2 целых числа и возвращает логическое значение.
  • В теле метода нужно вернуть результат решения задачи. Для вычисления результата нужно использовать поля класса (leftA, leftB) и значения из аргументов (rightA, rightB).
  • В случае вызова метода у некорректного интервала, нужно выбросить ислючение IllegalStateException.
  • В случае передачи в качестве аргумента некорректных значений интервала, нужно выбросить исключение IllegalArgumentException.

Проверьте корректность работы метода с помощью юнит-теста

  • В классе RangeSuite создайте публичный метод intersectsIntsTest. Он помечен аннотацией @Test, ничего не возвращает и не принимает аргументы.
  • В теле метода intersectsIntsTest нужно создать переменную типа Range и инициализировать ее экземпляром класса Range.
  • Далее нужно 6 раз вызвать метод intersects у экземпляра класса Range.
  • Перед каждым вызовом метода intersects нужно инициализировать поля экземпляра класса Range значениями from = leftA, to = leftB из разных тестов, описанных в задаче. В качестве аргументов метода intersects нужно передавать значения rightA, rightB.
  • Результат каждого вызова нужно указать в качестве единственного аргумента статического метода assertTrue или assertFalse класса Assert.
  • После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.

Проверьте юнит-тестами, что метод выбрасывает ислючение при некорректных данных

  • В классе RangeSuite создайте 2 публичных нестатических метода intersectsIntWrongLeftTest и intersectsIntWrongRightTest.
  • Пометьте их аннотацией @Test с элементом expected равным IllegalArgumentException.class
  • В теле метода intersectsIntWrongLeftTest инициализируйте экземпляра класса Range.
  • Заполните значения его полей так, чтобы этот экземпляр получился некорректным.
  • Вызовите у экземпляра класса метод intersects и передайте ему в качестве аргументов корректные значения интервала.
  • В теле метода intersectsIntWrongRightTest проверьте что метод hasIntersection выбрасывает исключение в случае, когда экземпляр класса корректный, а переданные аргументы - нет.
  • После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.
