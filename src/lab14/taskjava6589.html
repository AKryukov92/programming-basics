Задача

Реализуйте статический метод для решения задачи ЛР5#6589 (открыть в новой вкладке)

  • В папке для исходного кода создайте класс Range.
  • В классе Range реализуйте публичный статический метод hasIntersection.
  • Он принимает в качестве аргумента 4 целых числа и возвращает логическое значение.
  • В теле метода нужно вернуть результат решения задачи, используя значения аргументов в качестве исходных данных.
  • В случае передачи некорректных значений интервалов, нужно выбросить исключение IllegalArgumentException.

Проверьте корректность работы метода с помощью юнит-теста

  • В папке для юнит-тестов создайте класс RangeSuite.
  • В классе RangeSuite создайте публичный метод hasIntersectionIntsTest. Он помечен аннотацией @Test, ничего не возвращает и не принимает аргументы.
  • В теле метода hasIntersectionIntsTest нужно 6 раз вызвать статический метод hasIntersection класса Range. При каждом вызове в качестве аргументов нужно передавать значения из разных тестов, описанных в задаче.
  • Результат каждого вызова нужно указать в качестве единственного аргумента статического метода assertTrue или assertFalse класса Assert.
  • После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.

Проверьте юнит-тестами, что метод выбрасывает ислючение при некорректных данных

  • В классе RangeSuite создайте 2 публичных нестатических метода hasIntersectionIntsWrongLeftTest и hasIntersectionIntsWrongRightTest.
  • Пометьте их аннотацией @Test с элементом expected равным IllegalArgumentException.class
  • В теле метода hasIntersectionIntsWrongLeftTest вызовите метод hasIntersection и укажите такие значения, в которых левый интервал будет некорректным.
  • В теле метода hasIntersectionIntsWrongRightTest вызовите метод hasIntersection и укажите такие значения, в которых правый интервал будет некорректным.
  • После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.
