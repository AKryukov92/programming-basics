Задача

Реализуйте статический метод для решения задачи ЛР5#6589 (открыть в новой вкладке) используя экземпляры класса

  • Объявите у класса Range поля для целочисленных значений "from" и "to".
  • В классе Range реализуйте еще один публичный статичекий метод hasIntersection.
  • Новый метод hasIntersection принимает в качестве аргументов 2 экземпляра класса Range и возвращает логическое значение.
  • В теле метода нужно вернуть результат решения задачи. В вычислениях нужно использовать поля "from" и "to" экземпляров класса Range, которые были переданы в качестве аргументов.
  • В случае передачи некорректных интервалов, нужно выбросить ислючение IllegalArgumentException.

Проверьте корректность работы метода с помощью юнит-теста

  • В классе RangeSuite создайте публичный метод hasIntersectionRangesTest. Он помечен аннотацией @Test, ничего не возвращает и не принимает аргументы.
  • В теле метода hasIntersectionRangesTest нужно создать 2 переменных типа Range и инициализировать их экземплярами класса Range.
  • Далее нужно 6 раз вызвать статичекий метод hasIntersection класса Range.
  • Перед каждым вызовом метода hasIntersection нужно инициализировать поля "from" и "to" экземпляров класса Range значениями из разных тестов, описанных в задаче.
  • Результат каждого вызова нужно указать в качестве единственного аргумента статического метода assertTrue или assertFalse класса Assert.
  • После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.

Проверьте юнит-тестами, что метод выбрасывает ислючение при некорректных данных

  • В классе RangeSuite создайте 2 публичных нестатических метода hasIntersectionRangesWrongLeftTest и hasIntersectionRangesWrongRightTest.
  • Пометьте их аннотацией @Test с элементом expected равным IllegalArgumentException.class
  • В теле метода hasIntersectionRangesWrongLeftTest инициализируйте два экземпляра класса Range: left и right.
  • Заполните значения их полей так, чтобы левый интервал получился некорректным.
  • Вызовите метод hasIntersection и передайте ему left и right в качестве аргументов.
  • В теле метода hasIntersectionRangesWrongRightTest проверьте что метод hasIntersection выбрасывает исключение в случае, когда правый интервал некорректный.
  • После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.
