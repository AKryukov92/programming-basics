Задача

Реализуйте статический метод для решения задачи ЛР5#6589 (открыть в новой вкладке) используя экземпляры класса

  1. Объявите у класса Range поля для целочисленных значений "from" и "to".
  2. В классе Range реализуйте еще один публичный статичекий метод hasIntersection.
  3. Новый метод hasIntersection принимает в качестве аргументов 2 экземпляра класса Range и возвращает логическое значение.
  4. В теле метода нужно вернуть результат решения задачи. В вычислениях нужно использовать поля "from" и "to" экземпляров класса Range, которые были переданы в качестве аргументов.
  5. Если методу были переданы некорректные значения исходных данных, он должен выбросить исключение IllegalArgumentException, содержащий текст ошибки из задачи.

Проверьте корректность работы метода с помощью юнит-теста

  1. В классе RangeSuite создайте юнит-тест hasIntersectionRangesTest.
  2. В теле метода hasIntersectionRangesTest создайте 2 переменных типа Range: left и right. Инициализируйте их экземплярами класса Range.
  3. Для каждой строки тестовых данных из задачи напишите в теле юнит-теста фрагмент кода:
    1. Инициализируйте поля "from" и "to" переменной left значениями leftA и leftB.
    2. Инициализируйте поля "from" и "to" переменной right значениями rightA и rightB.
    3. Вызовите статический метод hasIntersection класса Range и передайте ему в качестве аргументов значения переменных left и right.
    4. Корректность результата вызова метода hasIntersection проверьте с помощью статических методов assertTrue или assertFalse класса Assert.
  4. После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.

Проверьте юнит-тестами, что метод выбрасывает ислючение при некорректных данных

  1. В классе RangeSuite создайте 2 юнит-теста hasIntersectionRangesWrongLeftTest и hasIntersectionRangesWrongRightTest.
  2. Они должны проверять факт возникновения исключения IllegalArgumentException.
  3. В теле метода hasIntersectionRangesWrongLeftTest инициализируйте два экземпляра класса Range: left и right.
  4. Заполните значения их полей так, чтобы левый интервал получился некорректным.
  5. Вызовите метод hasIntersection и передайте ему left и right в качестве аргументов.
  6. В теле метода hasIntersectionRangesWrongRightTest проверьте что метод hasIntersection выбрасывает исключение в случае, когда правый интервал некорректный.
  7. После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.
