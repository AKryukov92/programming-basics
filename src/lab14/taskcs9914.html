Задача

Реализуйте НЕстатический метод для решения задачи ЛР5#6589 (открыть в новой вкладке)

  1. Убедитесь, что у класса Range объявлены поля для целочисленных значений "from" и "to".
  2. В классе Point реализуйте публичный нестатический метод Intersects(без ключевого слова static перед типом возвращаемого значения).
  3. Метод Intersects принимает в качестве аргумента 2 целых числа и возвращает логическое значение.
  4. В теле метода нужно вернуть результат решения задачи. Для вычисления результата нужно использовать поля класса (leftA, leftB) и значения из аргументов (rightA, rightB).
  5. В случае вызова метода у некорректного интервала, нужно выбросить ислючение InvalidOperationException.
  6. В случае передачи в качестве аргумента некорректных значений интервала, нужно выбросить исключение ArgumentException.

Проверьте корректность работы метода с помощью юнит-теста

  1. В классе RangeSuite создайте юнит-тест IntersectsIntsTest.
  2. В теле метода IntersectsIntsTest создайте переменную left типа Range и инициализируйте ее экземпляром класса Range.
  3. Для каждой строки тестовых данных из задачи напишите в теле юнит-теста фрагмент кода:
    1. Инициализируйте поля "from" и "to" переменной left значениями leftA и leftB.
    2. Вызовите метод Intersects у экземпляра класса Range из переменной left.
    3. Передайте ему в качестве аргументов значения rightA, rightB.
    4. Добавьте выражение для проверки результата вызова метода Intersects.
  4. После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.

Проверьте юнит-тестами, что метод выбрасывает ислючение при некорректных данных

  1. В классе RangeSuite создайте юнит-теста IntersectsIntWrongLeftTest, который проверяет возникновение исключения InvalidOperationException, и юнит-тест IntersectsIntWrongRightTest, который проверяет возникновение исключения ArgumentException.
  2. В теле метода IntersectsIntWrongLeftTest инициализируйте экземпляра класса Range.
  3. Заполните значения его полей так, чтобы этот экземпляр получился некорректным.
  4. Вызовите у экземпляра класса метод Intersects и передайте ему в качестве аргументов корректные значения интервала.
  5. В теле метода IntersectsIntWrongRightTest проверьте что метод hasIntersection выбрасывает исключение ArgumentException в случае, когда экземпляр класса корректный, а переданные аргументы - нет.
  6. После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.
