Задача

Реализуйте НЕстатический метод для решения задачи ЛР5#6589 (открыть в новой вкладке)

  • Убедитесь, что у класса Range объявлены поля для целочисленных значений "from" и "to".
  • В классе Point реализуйте публичный нестатический метод intersects(без ключевого слова static перед типом возвращаемого значения).
  • Метод intersects принимает в качестве аргумента экземпляр класса Range и возвращает логическое значение.
  • В теле метода нужно вернуть результат решения задачи. Для вычисления результата нужно использовать поля экземпляра, у которого вызвали метод (from = leftA, to = leftB), и поля экземпляра, который передали в качестве аргумента (from = rightA, to = rightB).
  • В случае вызова метода у некорректного интервала, нужно выбросить ислючение IllegalStateException.
  • В случае передачи в качестве аргумента некорректного интервала, нужно выбросить исключение IllegalArgumentException.

Проверьте корректность работы метода с помощью юнит-теста

  • В классе RangeSuite создайте публичный метод intersectsRangeTest. Он помечен аннотацией @Test, ничего не возвращает и не принимает аргументы.
  • В теле метода intersectsRangeTest нужно создать 2 переменных типа Range и инициализировать ее экземпляром класса Range.
  • Далее нужно 6 раз вызвать метод intersects у экземпляра класса Range.
  • Перед каждым вызовом метода intersects нужно инициализировать поля экземпляров класса Range значениями из разных тестов, описанных в задаче.
  • Результат каждого вызова нужно указать в качестве единственного аргумента статического метода assertTrue или assertFalse класса Assert.
  • После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.

Проверьте юнит-тестами, что метод выбрасывает ислючение при некорректных данных

  • В классе RangeSuite создайте 2 публичных нестатических метода intersectsIntWrongLeftTest и intersectsIntWrongRightTest.
  • В теле метода intersectsIntWrongLeftTest проверьте что метод intersects выбрасывает исключение IllegalStateException в случае, когда метод вызван у некорректного экземпляра, а в качестве аргументов был передан корректный.
  • В теле метода intersectsIntWrongRightTest проверьте что метод intersects выбрасывает исключение IllegalArgumentException в случае, когда метод вызван у корректного экземпляра, а в качестве аргументов был передан некорректный.
  • После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.
