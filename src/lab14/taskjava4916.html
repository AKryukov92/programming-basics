Задача

Реализуйте НЕстатический метод для решения задачи ЛР5#6589 (открыть в новой вкладке)

  1. Убедитесь, что у класса Range объявлены поля для целочисленных значений "from" и "to".
  2. В классе Range реализуйте публичный нестатический метод intersects(без ключевого слова static перед типом возвращаемого значения).
  3. Метод intersects принимает в качестве аргумента экземпляр класса Range и возвращает логическое значение.
  4. В теле метода верните результат решения задачи. Для его вычисления используйте поля экземпляра, у которого вызвали метод (from = leftA, to = leftB), и поля экземпляра, который передали в качестве аргумента (from = rightA, to = rightB).
  5. В случае вызова метода у некорректного интервала, нужно выбросить ислючение IllegalStateException.
  6. В случае передачи в качестве аргумента некорректного интервала, нужно выбросить исключение IllegalArgumentException.

Проверьте корректность работы метода с помощью юнит-теста

  1. В классе RangeSuite создайте юнит-тест intersectsRangeTest.
  2. В теле метода intersectsRangeTest создайте 2 переменных типа Range: left и right. Инициализируйте их экземплярами класса Range.
  3. Для каждой строки тестовых данных из задачи напишите в теле юнит-теста фрагмент кода:
    1. Инициализируйте поля "from" и "to" переменной left значениями leftA и leftB.
    2. Инициализируйте поля "from" и "to" переменной right значениями rightA и rightB.
    3. Вызовите метод intersects у экземпляра класса Range из переменной left.
    4. Передайте ему в качестве аргументов переменную right.
    5. Добавьте выражение для проверки результата вызова метода intersects.
  4. После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.

Проверьте юнит-тестами, что метод выбрасывает ислючение при некорректных данных

  1. В классе RangeSuite создайте 2 публичных нестатических метода intersectsRangeWrongLeftTest и intersectsRangeWrongRightTest.
  2. В теле метода intersectsRangeWrongLeftTest проверьте что метод intersects выбрасывает исключение IllegalStateException в случае, когда метод вызван у некорректного экземпляра, а в качестве аргументов был передан корректный.
  3. В теле метода intersectsRangeWrongRightTest проверьте что метод intersects выбрасывает исключение IllegalArgumentException в случае, когда метод вызван у корректного экземпляра, а в качестве аргументов был передан некорректный.
  4. После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.
