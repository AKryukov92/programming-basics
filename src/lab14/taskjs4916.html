Задача

Реализуйте функцию для решения задачи ЛР5#6589 (открыть в новой вкладке) как поле объекта

  • Создайте юнит-тест для проверки задачи и подпишите его словами "проверка пересечения одного интервала с другим. Границы другого интервала заданы 1 объектом".
  • Инициализируйте объект с полями "a" и "b" в теле юнит-теста. Значения полей возьмите из 1 и 2 столбца исходных данных первого теста.
  • Инициализируйте у объекта поле intersects анонимной функцией, которая принимает 2 аргумента.
  • Реализуйте функцию intersects так, чтобы она возвращала результат решения задачи. В качестве исходных данных используйте поля "a" и "b" объектов. Первый интервал - этот же объект. Второй интервал передается в качестве аргумента функции intersects.
  • В теле юнит-теста инициализируйте второй объект с полями "a" и "b". Значения для полей возьмите из 3 и 4 столбика исходных данных первого теста.
  • Вызовите функцию intersects у первого объекта и передайте в качестве аргумента второй объект.
  • Проверьте корректность результатов вызова функции intersects с помощью функции expect в сочетании с функцией toBeFalsy или toBeTruthy.
  • Вызовите функцию intersects еще по одному разу для каждого оставшегося теста. Перед каждым вызовом нужно присвоить полям обоих объектов значения из соответствующего теста.
  • Проверьте что функция intersects выбрасывает исключение в случае с некорректными исходными данными. Воспользуйтесь функцией expect в сочетании с toThrow.
  • При открытии файла SpecRunner.html, в нем должна быть зеленая строка "проверка пересечения одного интервала с другим. Границы другого интервала заданы 1 объектом".
