Задача

Реализуйте статический метод для решения задачи ЛР5#6589 (открыть в новой вкладке)

  1. В папке для исходного кода создайте класс Range.
  2. В классе Range реализуйте публичный статический метод HasIntersection.
  3. Он принимает в качестве аргумента 4 целых числа и возвращает логическое значение.
  4. В теле метода верните результат решения задачи, используя значения аргументов в качестве исходных данных.
  5. Если методу были переданы некорректные значения исходных данных, он должен выбросить исключение ArgumentException, содержащий текст ошибки из задачи.

Проверьте корректность работы метода с помощью юнит-теста

  1. В папке для юнит-тестов создайте класс RangeSuite.
  2. В файле RangeSuite добавьте директиву using CodeProject;
  3. В классе RangeSuite создайте юнит-тест HasIntersectionIntsTest.
  4. В теле метода HasIntersectionIntsTest вызовите метод HasIntersection.
  5. Передайте в качестве аргументов значения 2, 3, 5 и 7.
  6. Проверьте, что метод HasIntersection вернул значение false.
  7. В теле метода HasIntersectionIntsTest вызовите метод HasIntersection еще раз.
  8. Передайте в качестве аргументов значения 11, 17, 13, 19.
  9. Проверьте, что при повторном вызове метод HasIntersection вернул значение true.
  10. После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.

Проверьте юнит-тестами, что метод выбрасывает ислючение при некорректных данных

  1. В классе RangeSuite создайте 2 юнит-теста HasIntersectionIntsWrongRedTest и HasIntersectionIntsWrongGreenTest.
  2. Они должны проверять факт возникновения исключения ArgumentException.
  3. В теле метода HasIntersectionIntsWrongRedTest вызовите метод HasIntersection и укажите такие значения, в которых левый интервал будет некорректным.
  4. В теле метода HasIntersectionIntsWrongGreenTest вызовите метод HasIntersection и укажите такие значения, в которых правый интервал будет некорректным.
  5. После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.
