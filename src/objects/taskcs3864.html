Задача

Реализуйте статический метод для решения задачи ЛР5#3864 (открыть в новой вкладке)

  1. В папке для исходного кода создайте класс Direction.
  2. В классе Direction объявите поля для действительных значений "angle" и "distance".
  3. В классе Library реализуйте публичный статический метод BackwardTask.
  4. Он принимает в качестве аргумента два экземпляра класса Point и возвращает экземпляр класса Direction.
  5. В теле метода вычислите горизонтальное проложение и дирекционный угол, используя значения аргументов в качестве исходных данных.
  6. Сформируйте ответ в виде экземпляра класса Direction и верните его в качестве результата.

Проверьте корректность работы метода с помощью юнит-теста

  1. В папке для юнит-тестов создайте класс GeodesySuite.
  2. В классе GeodesySuite создайте юнит-тест BackwardTaskTest.
  3. В теле юнит-теста объявите переменные src и dest типа Point и переменную direction типа Direction.
  4. Инициализируйте переменные экземплярами соответствующих классов.
  5. Для каждой строки тестовых данных из задачи напишите в теле юнит-теста фрагмент кода:
    1. Инициализируйте поля x и y переменной src значениями Xa и Ya.
    2. Инициализируйте поля x и y переменной dest значениями Xb и Yb.
    3. Вызовите статический метод BackwardTask класса Library.
    4. Передайте ему в качестве аргументов значения переменных src и dest.
    5. Результат вызова метода запишите в переменную direction.
    6. Проверьте, что значения полей distance и angle переменной dest соответствуют горизонтальному проложению и дирекционному углу соответственно.
  6. После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.
