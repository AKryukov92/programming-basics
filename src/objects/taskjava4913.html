Задача

Реализуйте статический метод для решения задачи ЛР2#4913 (открыть в новой вкладке)

  1. В папке для исходного кода создайте класс Direction.
  2. В классе Direction объявите поля для действительных значений "angle" и "distance".
  3. В классе Methods реализуйте публичный статический метод forwardTask.
  4. Он принимает в качестве аргумента экземпляр класса Point, экземпляр класса Direction и возвращает экземпляр класса Point.
  5. В теле метода вычислите координаты другой точки, используя значения аргументов в качестве исходных данных.
  6. Сформируйте ответ в виде экземпляра класса Point и верните его в качестве результата.
  7. Если методу был передан экземпляр класса Direction с некорректными значениями полей, метод должен выбросить исключение IllegalArgumentException, содержащий текст ошибки из задачи.

Проверьте корректность работы метода с помощью юнит-теста

  1. В папке для юнит-тестов создайте класс GeodesySuite.
  2. В классе GeodesySuite создайте юнит-тест forwardTaskTest.
  3. В теле юнит-теста объявите переменные src и dest типа Point и переменную direction типа Direction.
  4. Инициализируйте переменные экземплярами соответствующих классов.
  5. Для каждой строки тестовых данных из задачи напишите в теле юнит-теста фрагмент кода:
    1. Инициализируйте поля x и y переменной src значениями Xa и Ya.
    2. Инициализируйте поля distance и angle переменной direction значениями distance и angleDeg.
    3. Вызовите статический метод forwardTask класса Methods.
    4. Передайте ему в качестве аргументов значения переменных src и direction.
    5. Результат вызова метода запишите в переменную dest.
    6. Проверьте, что значения полей x и y переменной dest соответствуют значениям Xb и Yb в тесте.
  6. После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.

Проверьте юнит-тестами, что метод выбрасывает ислючение при некорректных данных

  1. В классе GeodesySuite создайте 3 юнит-теста forwardTaskNegativeDistance, forwardTaskZeroDistance, forwardTaskNegativeAngle.
  2. Они должны проверять факт возникновения исключения IllegalArgumentException.
  3. В теле юнит-теста forwardTaskNegativeDistance вызовите метод forwardTask и укажите такие аргументы, в которых горизонтальное проложение будет отрицательным.
  4. В теле юнит-теста forwardTaskZeroDistance вызовите метод forwardTask и укажите такие аргументы, в которых горизонтальное проложение будет равно 0.
  5. В теле юнит-теста forwardTaskNegativeAngle вызовите метод forwardTask и укажите такие аргументы, в которых дирекционный угол будет отрицательным.
