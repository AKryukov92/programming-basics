Задача

Реализуйте сеттер для полей класса Range

  1. Убедитесь, что у класса Range объявлены поля для целочисленных значений "from" и "to".
  2. В классе Range замените модификатор доступа к полям "from" и "to". Вместо "public" укажите "private".
  3. В классе Range реализуйте метод Init. Он принимает 2 целых числа и ничего не возвращает. В теле метода Init инициализируйте поля класса значениями из аргументов.
  4. Если методу были переданы некорректные значения исходных данных, он должен выбросить исключение ArgumentException, содержащий текст ошибки из задачи.
  5. Замените инициализацию полей класса на вызов метода Init в юнит-тестах:
    1. HasIntersectionRangesTest
    2. HasIntersectionRangesWrongLeftTest
    3. HasIntersectionRangesWrongRightTest
    4. IntersectsIntsTest
    5. IntersectsIntsWrongRightTest
    6. IntersectsRangeTest
    7. IntersectsRangesWrongRightTest
  6. Измените тип ожидаемого исключения с InvalidOperationException на ArgumentException в юнит-тестах:
    1. IntersectsIntWrongLeftTest
    2. IntersectsRangeWrongLeftTest
  7. После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.

Проверьте с помощью юнит-теста, что метод корректно выбрасывает исключение

  1. В классе RangeSuite создайте юнит-тест InitWrongTest.
  2. В теле юнит-теста создайте экземпляр класса Range.
  3. Вызовите метод Init и передайте ему в качестве аргументов значения 79 и 73.
  4. Проверьте, что в юнит-тесте возникло исключение ArgumentException.

Переопредилите стандартный метод Equals для проверки эквивалентности двух экземпляров класса Range.

  1. В классе Range создайте публичный метод Equals и добавьте перед типом возвращаемого значения ключевое слово override.
  2. Метод Equals должен возвращать логическое значение и принимать в качестве аргумента экземпляр типа Object.
  3. Метод Equals должен возвращать истину при сравнении экземпляра класса с самим собой.
  4. Метод Equals должен возвращать ложь и не выбрасывать исключений при сравнении экземпляра класса со значением null.
  5. Метод Equals должен возвращать ложь и не выбрасывать исключений при сравнении с экземпляром постороннего класса.
  6. Результат метода Equals должен зависеть от значения полей класса Range.
