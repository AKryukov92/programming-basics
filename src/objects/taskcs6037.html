Задача

Реализуйте статический метод для решения задачи ЛР5#6589 (открыть в новой вкладке) используя экземпляры класса

  1. Объявите у класса Range поля для целочисленных значений "from" и "to".
  2. В классе Range реализуйте еще один публичный статичекий метод HasIntersection.
  3. Новый метод HasIntersection принимает в качестве аргументов 2 экземпляра класса Range и возвращает логическое значение.
  4. В теле метода нужно вернуть результат решения задачи. В вычислениях нужно использовать поля "from" и "to" экземпляров класса Range, которые были переданы в качестве аргументов.
  5. Если методу были переданы некорректные значения исходных данных, он должен выбросить исключение ArgumentException, содержащий текст ошибки из задачи.

Проверьте корректность работы метода с помощью юнит-теста

  1. В классе RangeSuite создайте юнит-тест HasIntersectionRangesTest.
  2. В теле метода HasIntersectionRangesTest создайте 2 переменных типа Range: red и green. Инициализируйте их экземплярами класса Range.
  3. Инициализируйте поле "from" переменной red значением 23.
  4. Инициализируйте поле "to" переменной red значением 27.
  5. Инициализируйте поле "from" переменной green значением 29.
  6. Инициализируйте поле "from" переменной green значением 31.
  7. Вызовите статический метод HasIntersection класса Range и передайте ему в качестве аргументов значения переменных red и green.
  8. Проверьте, что метод HasIntersection возвращает false.
  9. После запуска тестового метода, он должен быть отмечен зеленым знаком в списке методов.

Проверьте юнит-тестами, что метод выбрасывает ислючение при некорректных данных

  1. В классе RangeSuite создайте 2 юнит-теста HasIntersectionRangesWrongRedTest и HasIntersectionRangesWrongGreenTest.
  2. Они должны проверять факт возникновения исключения ArgumentException.
  3. В теле метода HasIntersectionRangesWrongRedTest инициализируйте два экземпляра класса Range: red и green.
  4. Заполните значения их полей так, чтобы левый интервал получился некорректным.
  5. Вызовите метод HasIntersection и передайте ему red и green в качестве аргументов.
  6. В теле метода HasIntersectionRangesWrongGreenTest проверьте что метод HasIntersection выбрасывает исключение в случае, когда правый интервал некорректный.
  7. После запуска тестовых методов, они должны быть отмечены зеленым знаком в списке методов.
