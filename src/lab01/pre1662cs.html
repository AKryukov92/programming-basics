Символ на русской раскладке на английской раскладке

Шаг 2/6

К шагу 1 К вступлению

Команда вывода в консоль состоит из двух частей: технического оформления и сообщения. Более корректный термин - "строковый литерал". Одна команда - одно сообщение. Если нужно вывести несколько сообщений, напишите несколько команд или объедините сообщения в одно.

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Task1662 { class Program { static void Main(string[] args) { Console.Write("Ура! Заработало!"); } } }

Компьютер определяет начало и конец сообщения по кавычкам ". Из за того, что кавычки одинаковы, любая первая будет значить "начало сообщения", а следующая - "конец сообщения".

Что если в вашей команде вывода будет всего одна кавычка?

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Task1662 { class Program { static void Main(string[] args) { Console.Write("Ура! Заработало!); } } }

В этом случае компьютер обнаружит кучу ошибок:

  • ) expected
  • ; expected
  • Newline in constant

Обратите внимание, что символы ); в конце команды тоже подсвечены как часть сообщения.

Все, что нужно для исправления - поставить кавычку в конец сообщения, перед закрывающей круглой скобочкой. Как было первом фрагменте на этом шаге.

К шагу 2 К вступлению

А что делать, если нужно вывести двойную кавычку внутри сообщения?

Просто напишите эту кавычку вместе с обратным слэшем: \". Для каждой кавычки внутри сообщения - свой обратный слэш.

Символ на русской раскладке на английской раскладке
\ Вводится только на английской раскладке Вводить с шифтом. Ищите символы | \ / находится на разных клавиатурах по-разному. Может быть возле клавиш shift и backspace.

Добавьте к вашей программе вывод кавычек до и после слова "Заработало!".

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Task1662 { class Program { static void Main(string[] args) { Console.Write("Ура! \"Заработало\"!"); } } }

В результате получится:

Ура! "Заработало"!

Обратите внимание, что кавычка с обратным слэшем уже не считается концом сообщения. Если у вас снова возникают ошибки, рассмотренные на шаге 2, обозначьте конец сообщения еще одной двойной кавычкой.

Шаг 4/6

К шагу 3 К вступлению

С помощью обратного слеша можно оформить одно сообщение, которое займет две строки. Добавьте комбинацию \n как в примере.

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Task1662 { class Program { static void Main(string[] args) { Console.Write("Ура!\nЗаработало!"); } } }

В результате получится:

Ура! Заработало!

На самом деле этот символ находится в конце каждой строки в текстовых документах. Мы его вводим когда нажимаем Enter. Подробнее на википедии

Другой способ сделать то же самое - воспользоваться другой командой вывода в консоль.

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Task1662 { class Program { static void Main(string[] args) { Console.WriteLine("Ура!"); Console.Write("Заработало!"); } } }

Ура! Заработало!

С точки зрения компьютера, существенной разницы нет. Поэтому пишите так, как вам будет понятнее.

Шаг 5/6

К шагу 4 К вступлению

Если вам вдруг понадобится выводить сам символ обратного слэша, воспользуйтесь комбинацией \\. Каждые два обратных слэша подряд в результате дадут один символ в консоли. Например у такой программы:

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Task1662 { class Program { static void Main(string[] args) { Console.Write("\\\\\\\\"); } } }

Будет такой результат

\\\\

Концепция использования двух символов для получения одного, но особенного, называется экранирование символов. А комбинации символов - escape-последовательностями.

Шаг 6/6

К шагу 5 К вступлению

В первой части задачи нужно вывести вот такое сообщение:

+---+ | | | | +---+

Весь этот текст можно втиснуть в одно сообщение как в следующем примере

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Task1662 { class Program { static void Main(string[] args) { Console.Write("+---+\n| |\n| |\n+---+"); } } }

А можно расписать четырьмя командами:

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Task1662 { class Program { static void Main(string[] args) { Console.WriteLine("+---+"); Console.WriteLine("| |"); Console.WriteLine("| |"); Console.WriteLine("+---+"); } } }

В обоих вариантах результат будет таким:

+---+ | | | | +---+

Оба варианта верны. Как правило, у задач есть несколько вариантов решения. Однако не всегда все эти варианты одинаково оптимальны. Кроме того, некоторые решения задач нужны для иллюстрации каких-нибудь важных терминов.

Главное правило: решите хоть как-нибудь. Это лучше, чем ничего не сделать.

Рассмотренных фактов должно быть достаточно для решения задачи №1662 полностью.
