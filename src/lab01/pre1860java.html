Шаг 1/5

К вступлению

Прежде чем что-то сохранять в переменную, нужно ее объявить. Ниже добавлена команда объявления переменной с именем "data". Слово "String" перед именем означает, что эта переменная может хранить в себе строку текста. Другими словами, это команда объявления строковой переменной "data". String это ее тип.

package lab01; import java.util.Scanner; public class Task1860 { public static void main(String[] args) { Scanner scan = new Scanner(System.in); String data; } }

После объявления переменной, можно получить информацию от пользователя и сохранить ее. Это действие оформляется следующим образом.

package lab01; import java.util.Scanner; public class Task1860 { public static void main(String[] args) { Scanner scan = new Scanner(System.in); String data; data = scan.nextLine(); } }

Когда пользователь запустит программу, введет какой-нибудь текст и нажмет Enter, программа получит текст от пользователя и запишет его в переменную data.

Обратите внимание, что в команде вывода переменной в консоль, мы не пишем ". Если несколько раз запустить эту программу и ввести в консоль разную информацию, то мы получим разный результат. Двойными кавычками оформляется постоянное сообщение, которое не меняется от вводимых данных.

К шагу 2 К вступлению

По заданию требуется вывести сообщение, состоящее из двух частей: постоянного фрагмента "Мы стремимся к " и изменяющегося фрагмента, который вводится пользователем. Чтобы выполнить задание, достаточно просто добавить команду вывода постоянного фрагмента.

package lab01; import java.util.Scanner; public class Task1860 { public static void main(String[] args) { Scanner scan = new Scanner(System.in); String data; data = scan.nextLine(); System.out.print("Мы стремимся к "); System.out.print(data); } }

Эту задачу можно решить другим способом. В нем постоянный и изменяющимся фрагменты склеивается с помощью символа +. Результат выводится одной командой.

package lab01; import java.util.Scanner; public class Task1860 { public static void main(String[] args) { Scanner scan = new Scanner(System.in); String data; data = scan.nextLine(); System.out.println("Мы стремимся к " + data); } }

В обоих случаях получится одинаковый результат

знаниямМы стремимся к знаниям

Запустите программу несколько раз. Посмотрите на результаты при вводе других значений из тестов, которые находятся в условии задачи.

Старайтесь избегать склеивания между собой фиксированных фрагментов текста. Так программа выглядит сложнее, чем на самом деле. Вот такой код

package lab01; import java.util.Scanner; public class Main { public static void main(String[] args) { Scanner scan = new Scanner(System.in); String x; x = scan.nextLine(); String y; y = scan.nextLine(); System.out.println("<point " + "x=" + "\"" + x + "\"" + " y=" + "\"" + y + "\"" + ">"); } }

Можно переписать вот так:

package lab01; import java.util.Scanner; public class Main { public static void main(String[] args) { Scanner scan = new Scanner(System.in); String x; x = scan.nextLine(); String y; y = scan.nextLine(); System.out.println("<point x=\"" + x + "\" y=\"" + y + "\">"); } }

Второй вариант выглядит проще, а результат никак не изменился

К вступлению

Вступление

В предыдущем разборе теории (открыть в новой вкладке) была рассмотрена команда вывода сообщения на экран. Однако полезные программы, которые занимаются исключительно выводом данных - большая редкость. Чтобы быть действительно полезной, программа должна быть универсальной - решать множество похожих задач. Задачи могут отличаться значениями в исходных данных.

Есть несколько разных способов передать программе исходные данные. Некоторые из них будут рассмотрены в следующих лабораторных. Мы начнем с получения информации через консоль. Этот способ проще всего оформлять. Это важно для разбора нескольких вводных тем.

Для считывания данных в программе нужно подготовить специальный инструмент, который называется Scanner. Читается как "сканер".

package lab01; import java.util.Scanner; public class Task1860 { public static void main(String[] args) { Scanner scan = new Scanner(System.in); } }

Исторически сложилось, что существует много инструментов, которые называются Scanner. Поэтому в начале программы мы добавили импорт именно того сканера, который нам нужен. В коде самой программы был создан сканер с именем scan.

Команда для получения данных от пользователя выглядит вот так

package lab01; import java.util.Scanner; public class Task1860 { public static void main(String[] args) { Scanner scan = new Scanner(System.in); scan.nextLine(); } }

Когда компьютер начнет выполнять ее, работа программы приостановится. С этого момента пользователь может ввести какой-нибудь текст. Когда он нажмет клавишу Enter, то введенный текст будет получен программой и она продолжит свою работу.

С помощью такой команды, программа просто получит текст. Если оформить ее именно так, то ничего полезного с информацией сделать нельзя. Программа сразу же забудет полученные данные.

Если мы хотим использовать эту информацию, ее нужно сохранить. С этой целью используют переменные.
