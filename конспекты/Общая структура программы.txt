Программа состоит из следующих частей:
* Подготовка к работе (общее оформление)
* Объявление и инициализация переменных для работы
* Получение данных
* Проверка данных
* Выполнение действий для решения задачи
* Вывод результата
* Завершение работы

Общее оформление зависит от языка.
Получение данных зависит от языка и технологии.
Завершение работы зависит от языка и технологии.


Условия используются если в программе может быть два варианта развития событий.
На практике условия используются:
* для проверки исходных данных
* При выборе выполняемых действий из двух вариантов

Простейшее условие состоит из трех частей:
* Левое выражение
* Оператор сравнения
* Правое выражение
Левое и правое выражение могут быть:
* Числовыми константами
* Переменными
* Арифметическими выражениями
* Вызовами функций
Программа сначала вычислит значения левого и правого выражения, а затем применит оператор сравнения.
Оператор сравнения:
* < 
* > 
* <=
* >=
* == Истинно, если левое меньше правого


Guard-выражения (Гвард-выражения) - условная конструкция, которая проверяет корректность исходных данных.
Гвард-выражения помещаются сразу после получения исходных данных.
Пишется в виде:
 if (условие) {
	 Сообщение об ошибке
	 return;
 }

Выбор выполняемых действий пишется в виде:
if (условие) { //Условие от которого зависит выбор
	//Первый вариант действий
} else [
	//Второй вариант действий
]

Действия могут содержать дополнительные вложенные условия.

 Точки остановки


 
При возникновении ошибки компиляции:
Если ошибка выдает какое-либо сообщение, нужно перевести его.
Если ошибка появилась после недавних изменений, нужно проверить:
* Правильность написания операторов
* Расстановку знаков пунктуации

При возникновении ошибки в ходе работы программы, проверить:
* Всегда ли переменным присваиваются начальные значения
* Присутствует ли деление на ноль

Если программа выдает неожиданные результаты:
* Проверить операторы ввода и вывода
* Определить все случаи (тесты, наборы исходных данных) при которых возникает ошибка
* Выявить закономерность
* Определить какие ветви выполнения кода вызывают ошибку
* Проверить логику работы программы в проблемных ветвях кода

Если исходный код программы слишком сложный, чтобы найти ошибку:
* Проверить оформление задачи, разделить ее на составные части
* Выделить повторяющиеся действия в функции (методы) и проверить их

Если задача слишком сложная чтобы решить:
* Выразить исходную задачу в терминах программирования
* Разбить исходную задачу на несколько подзадач
* Определить какие из подзадач решались ранее
* Написать и проверить функции (методы) для решения новых подзадач
* Воспользоваться проверенными функциями (методами) для решения основной задачи
