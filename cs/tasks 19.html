Подготовка проекта

  1. Создайте новое решение Stage2 типа "консольное приложение .NET Framework". Имя проекта укажите как CodeProject.
  2. Убедитесь, что в проекте CodeProject присутствует класс Program. Он автоматически создается вместе с проектом типа "консольное приложение".
  3. Добавьте в проекте CodeProject класс Library. Он будет содержать методы для решения задач. Напишите слово public перед фразой class Library.

Задача

Пользователь вводит слово (goal). Нужно напечатать на экран фразу "Мы стремимся к " и затем слово от пользователя.
  1. В классе Library реализуйте публичный статический метод task1860.
  2. Он принимает в качестве аргумента 1 строку.
  3. В теле метода task1860 напечатайте в консоль решение задачи, используя значение аргумента в качестве исходных данных.
Код метода будет выглядеть так:
public class Library { public static void task1860(String value) { System.out.println("Мы стремимся к " + value); } }
В методе main класса Program добавьте следующий код:
//вызов метода task1860 из класса Library Library.task1860("знаниям");//литерал "знаниям" передан в качестве аргумента String goal = "цели"; //значение переменной goal передано в качестве аргумента Library.task1860(goal);//внутри метода это значение будет доступно под именем value Library.task1860("результату");
В результате запуска метода main класса Program, в консоли должен появиться текст:
Мы стремимся к знаниям Мы стремимся к цели Мы стремимся к результату

Реализуйте метод для решения задачи ЛР5#5662(открыть в новой вкладке)

  3. В теле метода верните результат решения задачи, используя значения аргументов в качестве исходных данных.

Проверьте корректность работы метода с помощью юнит-теста

  1. В классе TestSuite создайте публичный метод Task5662test (далее "юнит-тест"). Добавьте ему атрибут TestMethod. Сам метод ничего не возвращает и не принимает аргументы.
  2. В теле юнит-теста 5 раз вызовите статический метод Task5662 класса Library. Каждый раз в качестве аргументов нужно передавать значения из разных тестов, описанных в задаче.

  4. Первый аргумент метода AreEqual это ожидаемое значение. Второй аргумент - результат вызова метода task5662. Третий аргумент (точность) в этом случае не указывается. Она нужна только для проверки действительных чисел.
  5. После запуска юнит-теста, он должен быть отмечен зеленым знаком в списке методов.
