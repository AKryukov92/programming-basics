using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Example { class Program { static void Main(string[] args) { String data; data = Console.ReadLine(); double x; x = double.Parse(data); double result; Console.Write("Результат: {0:F4}", result); } } }

Следующим шагом нужно оформить вычисление результата. Команда выглядит вот так:

result = x + 7;

Выражение "result = ..." как и раньше это "присвоить переменной result значение ...". А значение получается при сложении переменной x и числа 7.

Предположите, что получится, если вычисление оформить так:

data = x + 7;

Или так:

result = data + 7;

Шаг 2/3

К шагу 1 К вступлению

Верный вариант решения этой задачи выглядит так:

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Example { class Program { static void Main(string[] args) { String data; data = Console.ReadLine(); double x; x = double.Parse(data); double result; result = x + 7; Console.Write("Результат: {0:F4}", result); } } }

Отдельные части арифметического выражения названы терминами: операторы и операнды. Оператор это действие, а операнд - то над чем оно выполняется. Если отметить их цветом в решаемой задаче, то получится так:

result = x + 7; Строго говоря, "result" это тоже операнд, а "=" это тоже оператор.

В языке поддерживается множество операторов. На данный момент нам интересны следующие:

*умножение /деление +сложение чисел или склеивание строк -вычитание

Операнды намного более разнообразны. Ими могут быть:

  • Литералы. Т.е. числа, написанные в коде.
  • Переменные.
  • Выражения в круглых скобках.
  • Вызовы методов. Такие операторы будут рассмотрены позже.

Как правило, операнды должны быть совместимы с операторами. Например "+" можно использовать для работы со строками, а "-", "*" и "/" нельзя. Строки и числа можно склеить в одну строку оператором "+", но оператор "*" в том же выражении приведет к синтаксической ошибке.

Как вы думаете, есть ли подобные ограничения при работе с целыми и дробными (действительными) числами?

К шагу 2 К вступлению

Выражения можно составлять из множества операндов и операторов. Например так:

distance = initialVelocity * time + (acceleration * time * time) / 2;

Выражения с несколькими операторами компьютер разбивает на несколько шагов и затем последовательно их выполняет. Сначала вычисляются выражения в скобках. Затем порядок определяется приоритетом операторов. Например умножение и деление вычисляются раньше сложения и вычитания.

Фрагмент выше можно преобразовать в следующий набор команд без потери смысла:

step0 = acceleration * time; step1 = step0 * time; step2 = step1 / 2; step3 = initialVelocity * time; distance = step3 + step2;

Это просто иллюстрация действий компьютера. Такая запись - крайность, которая редко встречается в реальном коде.

Подробную информацию о приоритете операторов можно найти в документации.

Будьте внимательны при работе с натуральными дробями. Одна из самых распространенных ошибок происходит во время преобразования выражений вроде такого:

d= a + b c

По правилам вычислений таких выражений, сначала нужно сложить "a" с "b". А на "c" нужно делить полученную сумму. Если потерять бдительность, то получится:

d = a + b / c;

Но это не правильно. В этом случае компьютер сначала разделит "b" на "c" и затем уже прибавит "a". Такой порядок действий определен приоритетом операторов в языке.

Правильный вариант выглядит так:

d = (a + b) / c;

В нем были использованы скобочки, чтобы комьютер сначала сложил числа "a" с "b" и их сумму разделил на "c".

Другой правильный вариант выглядит так:

sum = a + b; d = sum / c;

Для компьютера нет существенной разницы между этими двумя способами. Дополнительные переменные вводят для того, чтобы упростить чтение сложных вычислений человеком. Это важно, потому что код чаще читают, чем пишут.

Вступление

В предыдущем разборе теории (открыть в новой вкладке) был рассмотрен ввод чисел с клавиатуры.

Сейчас будет разобран пример оформления вычислений в программе.

Прежде чем отдавать компьютеру команду "посчитай результат", нужно понять назначение каждой переменной.

Какие переменные были объявлены в фрагменте программы ниже?

using System; using System.Collections.Generic; using System.Linq; using System.Text; using System.Threading.Tasks; namespace Example { class Program { static void Main(string[] args) { String data; data = Console.ReadLine(); double x; x = double.Parse(data); } } }

Их всего две:

  • data - нужна чтобы получить значение от пользователя в текстовом виде.
  • x - нужна для хранения значения от пользователя в виде числа (действительного)

Для хранения результата будет полезно создать еще одну переменную. В абстрактной задаче ее можно назвать просто "result". Если же в задаче, к примеру, дата плотность тела и его объем, а нужно найти массу, то переменным нужно дать более осмысленные названия: density, volume, mass.

Добавьте команду объявления переменной для результата и команду для вывода ее в консоль с нужной точностью.
